hexo next 添加不蒜子访客记录及gitalk评论功能

Posted on 2020-05-08 Edited on 2020-05-26 In 杂项 Views:
详细的配置过程与经验记录

首先贴一下我的hexo相关环境：

  • hexo-cli: 3.1.0
  • node: 12.5.0
  • Next: 7
  • os: Darwin 19.3.0 darwin x64

吐槽一句，hexo next的不同版本间差异比较明显，见过太多博客不提版本直接上教程，看完反而更迷惑。。。如果以上环境和你的差别较大，就可以考虑关掉这篇博客了，或者升级/降级至以上版本。

在以上版本中，不蒜子统计和gitalk已经预先集成在了hexo next主题中，不需要自行修改或新建swig文件！在next主题的配置文件中分别搜索busuanzi 与 gitalk（注意只有一个t），然后将enable改为true即可。至此不蒜子的配置就结束了，当然如果需要定制效果就需要做其他修改，可以参考其他博客。gitalk则要稍微复杂一点，下面详细介绍。

首先，gitalk与同样应用广泛的gitment一样，并不依赖于第三方平台，而是将每篇文章的评论自动转化为git仓库中的一个issue，因此相比第三方平台稳定性较高（国内的平台自身容易出问题，国外的连接麻烦）。

既然是通过issue来管理评论，那评论者肯定要是合法的github账户。为实现这一目的，我们需要去github上注册一个oauth application（一个标准的授权协议）。注册链接

其中有三个必填栏目：

  • Application name： 随便填，只要自己记得住；
  • Homepage URL： 填博客页面地址，比如github pages的话就是https://.github.io （https不能省略）；
  • Authorization callback URL：同上，一样填成博客地址的就行。

然后就可以看到Client ID和Client Secret，后面会用到。

编辑next的配置文件，搜索gitalk，修改以下内容：

  • githubID -> 你的用户名；
  • repo ->仓库名，可以是博客仓库也可以是其他仓库，比如新建一个空仓库专门保存博客评论，注意这里填的是博客名，比如仓库为abcd.git，这里就要填abcd；
  • ClientID -> oauth 里的Client ID；
  • ClientSecret -> oauth里的Client Secret；
  • adminUser -> 填任何用户的ID都行，不过一般要有自己；

然后保存，hexo重新生成，提交就ok了。需要注意的是，这两个功能都需要网络，所以本地hexo show看到的效果和真实的不一定一样，gitalk会无法使用，而不蒜子会显示非常大的访问量，这除了会带来短时间的喜悦与发现真相后的自闭外没有任何意义。。。所以具体效果一定要hexo deploy后再看。

最后，每篇文章的gitalk评论区都需要adminUser范围内的人手动初始化，否则无法评论。这一点在文章多了之后可能会比较麻烦，记得已经有人写过自动化工具了，不过我还没试过，感兴趣的话可以自己去搜索一下试一试。

其他就没啥了，本文参考内容较多，具体可行的步骤也是自己一点一点边查文档查博客边出来的，恕不能一一列举，不过gitalk配置部分主要参考了这篇文章：https://asdfv1929.github.io/2018/01/20/gitalk/ 。
